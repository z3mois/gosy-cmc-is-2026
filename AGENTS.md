# Памятка для работы с конспектом к госэкзамену

Этот файл нужно читать перед любыми правками в этой папке.

## Цель

Подготовить единый LaTeX-конспект по билетам к госэкзамену. Материал должен быть пригоден для повторения: не слишком краткий, но без лишней воды.

## Стиль ответа в билетах

- Писать по-русски, академично, но простыми формулировками.
- Начинать каждый билет с короткого плана ответа.
- По возможности держать в билете две полезные формы: связный конспект для чтения и формулировки, которые можно воспроизвести в письменном ответе.
- Давать определения, затем свойства, затем примеры и типовые сравнения.
- Для понятийных билетов явно разводить близкие термины и типичные смешения, например абстракция/инкапсуляция, перегрузка/переопределение, статический/динамический полиморфизм.
- Для алгоритмов указывать идею, вход/выход, основные шаги, сложность и ограничения.
- Для моделей ИИ и МО указывать постановку задачи, обучение/вывод, метрики качества и типичные ошибки.
- В конце билета держать блок "Что важно проговорить на экзамене".
- Не перегружать билет подробностями из соседних вопросов: оставлять краткую связку, а детальный механизм раскрывать в том билете, где он является основной темой.

## LaTeX

- Главный файл: `gosy.tex`.
- Каждый вопрос лежит отдельно в `tickets/ticketXX.tex`.
- Не писать преамбулу внутри отдельных билетов.
- Использовать стандартные окружения: `itemize`, `enumerate`, `description`.
- Короткие примеры кода помещать в `verbatim`, если не нужны подсветка или сложное форматирование.
- Формулы добавлять там, где они помогают быстро вспомнить суть.
- Не перегружать билет длинными листингами кода.

## Рабочий процесс

- Если пользователь просит "сократить", сначала сохранить смысловые опорные точки.
- Если пользователь просит "подробнее", добавлять объяснение через пример или сравнение.
- Если пользователь приносит готовый текст билета, встраивать его в соответствующий `ticketXX.tex`, не ломая структуру.
- Если в принесенном тексте есть полезный, но слишком подробный материал, переносить в текущий билет только экзаменационно нужное ядро, а остальное отмечать как материал для соседнего билета.
- Если есть сомнения по термину, отмечать место как `TODO` и предлагать уточнение.

## Разбор вопросов пользователя

- Если пользователь задает вопрос по конкретному билету, сначала читать соответствующий `tickets/ticketXX.tex`, а затем отвечать с опорой на текущий текст конспекта.
- По умолчанию отвечать именно в режиме чата: точечно, понятно и без автоматического расширения файлов.
- Объяснять так, чтобы пользователь мог понять идею и затем воспроизвести ее на экзамене: коротко сформулировать суть, затем дать пояснение, пример и при необходимости возможную формулировку для ответа.
- Не расписывать тему подробно без необходимости: глубину объяснения подбирать по вопросу пользователя.
- Если вопрос показывает, что в билете не хватает важного пояснения, примера или различения терминов, сначала отметить это в ответе. Вносить правку в `ticketXX.tex`, когда пользователь просит обновить файл или когда исправление очевидно полезно для конспекта.
- Не переписывать билет при каждом вопросе автоматически: сначала помогать разобраться, а править файл только при явной пользе.
- После существенных правок в билетах пересобирать `gosy.tex` и оставлять `gosy.pdf` актуальным.
- Помнить, что каждый билет в итоговом PDF должен начинаться с новой страницы; технически это задается в `gosy.tex` через `\includeticket`.
